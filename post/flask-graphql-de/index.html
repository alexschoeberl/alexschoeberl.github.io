Erstellen eines GraphQL-APIs mit Flask

Tutorial · Jun 12, 2019

GraphQL ist eine bei Facebook entwickelte Data-Query-Language, ähnlich SQL, die allerdings auf HTTP aufsetzt und der Kommunikation zwischen Webapplikationen bzw. Webclients dient. Sie stellt damit eine flexiblere Alternative zu statischen REST-APIs dar. In diesem Tutorial zeige ich, wie man mit dem Flask web application framework Daten über ein GraphQL-Interface zugänglich macht. Dabei wird bewusst auf den Einsatz von ORM-Frameworks und -Modellen verzichtet.

Um das Tutorial durchführen zu können, müssen einige Python-Packages installiert werden. Dafür kann folgender Befehl genutzt werden:

python3 -m pip install -U Flask-GraphQL graphene

Die zu entwickelnde Applikation wird Informationen über Bücher in Bibliotheken bereitstellen. Die folgende Auflistung zeigt die Struktur des Projektordners. Der gesamte Code des Tutorials findet sich in einem Github-Repository.

  • run.py
  • library/
    • __init__.py
    • setup.py
    • data.py
    • schema/
      • __init__.py
      • items.py
      • libraries.py
      • queries.py

Die Daten, die im Zuge dieses Tutorials zugänglich gemacht werden sollen, sind in der Datei data.py zu finden. Dabei handelt es sich um eine einfache Liste an Büchern, Buchreihen und CDs, die jeweils einer von zwei Bibliotheken zugeordnet sind. Das Web-API soll dabei nur jeweils die Elemente einer spezifizierten Bibliothek anzeigen.

In diesem Tutorial kommt das Graphene Framework zur Anwendung, das GraphQL Abfragen interpretieren und beantworten kann. Dafür muss ein Schema definiert werden, das der Struktur der Daten entspricht, die zugänglich gemacht werden sollen. Dieses Schema wird durch Python-Klassen und Klassenvariablen in Objekt-orientierter Weise repräsentiert. Die Module im Ordner schema enthalten die Definitionen dieser Klassen.

class Item(graphene.Interface):
			  id = graphene.ID(required=True)
			  title = graphene.String(required=True)
			  included = graphene.types.datetime.Date(required=True)


			class Book(graphene.ObjectType):
			  class Meta:
			    interfaces = (Item,)

			  author = graphene.String(required=True)
			  editor = graphene.String(required=True)
			  publisher = graphene.String(required=True)
			  published = graphene.Int(required=True)


			class Collection(graphene.ObjectType):
			  class Meta:
			    interfaces = (Item,)

			  books = graphene.List(graphene.NonNull(Book))
			  publisher = graphene.String(required=True)


			class CD(graphene.ObjectType):
			  class Meta:
			    interfaces = (Item,)

			  artist = graphene.String(required=True)
			  label = graphene.String(required=True)
			  published = graphene.Int(required=True)

Im Gegensatz zur flachen Hierarchie der Typen in der Datei data.py kommt bei der Definiton des Schemas Vererbung zum Einsatz. Um diese zu implementieren wird der graphene.Interface-Typ genutzt. Attribute, die als Teil des Interfaces definiert sind, stehen auch in allen Klassen, die dieses Interface implementieren, zur Verfügung. Das gilt zumindest, wenn man auf Objekte dieser Klassen mittels GraphQL zugreift. An den Definitionen der Attribute ist auch zu sehen, das GraphQL eine typisierte Sprache ist.

Nach der Definition des Schemas müssen die erstellten Klassen mit konkreten Daten befüllt werden. Dafür muss Graphene für jedes Attribut mitgeteilt werden, wie es aufgelöst werden soll. Direkt in der Klassendefiniton muss für jedes Attribut daher eine statische Methode mit der Signatur resolve_<attribute>(parent, info) implementiert werden. Der Parameter parent referenziert in den Beispielen hier immer das aktuelle Objekt, hat also eine ähnliche Funktion wie der Parameter self. Mittels parent.id kann daher die Identifikationsnummer des aktuellen Objekts herausgefunden und der gesuchte Wert aus den Daten ausgelesen werden. Resolver werden dabei nur ausgeführt, wenn der Wert des jeweiligen Attributs im Query angefragt wurde.

class Book(graphene.ObjectType):
			  def resolve_publisher(parent, info):
			    return items[parent.id].publisher

Entspricht eine Klassenvariable dem Listentyp, muss der Resolver auch eine Liste zurückgeben. Das ist zum Beispiel bei der Variable books der Collection-Klasse der Fall. Der im folgenden Codeausschnitt definierte Resolver liefert eine Liste aller Bücher zurück, die Teil einer Buchreihe sind. Da Buchreihen in den Daten nur Bücher und keine CDs enthalten, muss der Typ des Elements nicht überprüft werden.

class Collection(graphene.ObjectType):
			  def resolve_books(parent, info):
			    collection = items[parent.id]
			    return [Book(id=id) for id in collection.parts]

Ein ähnlicher Resolver findet sich auch in der Library-Klasse in der Datei libraries.py. Dort werden alle Elemente in den Daten gesucht, die Teil einer spezifizierten Bibliothek sind. In diesem Fall muss allerdings der richtige Typ des jeweiligen Elements ausgewählt werden. Die Klasse Query in der Datei queries.py stellt quasi den Einstiegspunkt einer GraphQL-Abfrage dar. In dieser Klasse wurde definiert, dass sowohl eine Liste aller Bibliotheken, als auch Informationen über eine spezifizierte Bibliothek abgerufen werden können. Um Informationen zu einer Bibliothek zu erhalten, muss ein String-Parameter mit der Bezeichnung name angegeben werden. Dieser Parameter wird an den Resolver weitergereicht.

class Query(graphene.ObjectType):
			  libraries = graphene.List(Library)
			  library = graphene.Field(Library, name=graphene.String())

			  def resolve_libraries(parent, info):
			    libraries = {items[id].library for id in items}
			    return [Library(name=name) for name in libraries]

			  def resolve_library(parent, info, name):
			    if name in {items[id].library for id in items}:
			      return Library(name=name)

In der Datei setup.py wird das erzeugte Schema geladen und der API-Endpunkt bei der Flask-Applikation registriert. Der Parameter graphiql bewirkt, dass eine interaktive Webapplikation bereitgestellt wird, über die das API im Browser abgefragt werden kann. Dafür muss der URL des Endpunkts mit dem Browser aufgerufen werden.

def add_interface(app):
			  from library.schema.queries import Query
			  from library.schema.items import Book, CD, Collection

			  schema = graphene.Schema(query=Query, types=[Book, Collection, CD])
			  view = GraphQLView.as_view('api', schema=schema, graphiql=True)
			  app.add_url_rule('/api', view_func=view)

Wird die Flask-Applikation jetzt gestartet, und die interaktive Webapplikation aufgerufen, können die Daten abgefragt werden. Dafür müssen GraphQL-Queries formuliert werden. Diese Abfragen beschreiben im Grunde genommen einfach, welche Attribute zurückgeliefert werden sollen. Handelt es sich bei einem Attribut um eine Liste, muss an dieser Stelle ein Sub-Query formuliert werden.

Zwei Queries und die Antwort auf Query2

Im linken Textfeld dieser Abbildung wurden zwei Queries definiert. Queries tragen einen Namen. Mittels der ersten Abfrage sollen die Namen aller verfügbaren Bibliotheken erfragt werden. Dafür wird das Attribut libraries der Query-Klasse abgefragt. Da es sich dabei um einen Listentyp handelt, muss ein Sub-Query angegeben werden. Von jedem Objekt der Library-Klasse soll nur der Name abgefragt werden. Der zweite Query fragt alle Elemente der Bibliothek mit dem Namen Library 1 ab. Das Feld __typename ist dabei ein standardmäßig vorhandenes Attribut, das den Typ eines Elements angibt. In diesem Query ist ein Sub-Query spezifiziert, der nur für Elemente des Typs Book ausgeführt wird. Im rechten Textfeld der Abbildung ist die Antwort auf diesen zweiten Query dargestellt.
