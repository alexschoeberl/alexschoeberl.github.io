A Learning Home

Blog · Feb 24, 2019

During the last year I set up a number of home automation systems using the Pabiana framework I developed. The framework is still work in progress. Those home automation systems have a number of inputs like cameras, UHF radio receivers or a list of devices currently connected to the WIFI network. They also have control over some external devices like lights, a hi-fi system or window shutters. The algorithms controlling these devices are considerate of their environment and therefore somewhat intelligent. However, they are manually programmed and thus not capable of learning and adapting.

Programming all those rules that control a smart home can get very complex. Average users should not have to learn to do that when using the framework. Of course, some of the more safety-relevant rules will have to be hard-coded. But the system should be able to learn the other ones. That's why I started to think about how this feature could be implemented.

The Path to a Learning System

There are three ways a system can learn things. It can learn by itself (unsupervised learning), through trail and error (reinforcement learning) or with specific targets (supervised learning), respectively requiring more user input and preparation. Reinforcement learning (RL) seems to be the right choice for this problem since it lets the user basically train the system through minor inputs. An additional input to the home automation system could be implemented for the user to rate system actions positively or negatively. There are, however, some problems with this setup. The first one is that reinforcement learning needs a lot of time and users will quickly get annoyed if the system keeps turning the lights and music on and off, waiting for praise. In the beginning the system will just take random actions. It is impractical to wait for days until it accidentally takes the right action given the circumstances. Reinforcement learning needs a lot of data and is therefore usually done within a simulated environment. Real-time is just to slow to generate enough interesting data in a reasonable amount of time. Also, the learning process for RL can be unstable if the rewards are sparse.

So what could be the right path to create a learning home automation system? It seems that multiple techniques have to be applied to solve the mentioned problems, including imitation learning, curiosity-driven learning with intrinsic rewards and model-based learning.

Imitation learning

Imitation learning is actually a form of supervised learning. In a first step data is collected from an expert by recording his actions. That data is then used to train the system. In the case of home automation, the user is the expert. The system will record the states through its inputs and the users responses to them. Using those records we can directly train the decision making policy. A different approach could be to define a high reward for imitating the user and train the system with these rewards. After some time the users might notice the system taking actions right before they were about to do them.

Curiosity-driven Learning

Imitation learning should be fairly stable at replicating user behaviour. However, user interaction might be limited to a few instances a day. Also, a smart home should be able to learn to control devices on its own. Curiosity-driven learning could be used to achieve that. An intrinsic reward function could be defined that incentivises the exploration and discovery of new states. Exploring states recorded for imitation learning could be particularly rewarded. The system would learn useful sequences of actions through this process. To prevent the system from being overly annoying, curious behaviour should only be intrinsically rewarded during specific periods of playtime. Playtime rewards have to become smaller over time to further encourage getting work done. Rewards provided by the user must have a notably larger impact.

Model-based learning

For a stable reinforcement learning process the learning rate has to be set to a rather small value. The system should only take baby steps. That, however, results in a long training time, especially in a real-world setting. Having an internal model of the environment to train the system on can speed up the process. In practice, the model will be a neural network that predicts the future state and resulting rewards based on the current state and action taken. In our case the model learns to imitate the homes behaviour. It is trained through supervised learning with recorded data. Training the system on the model has the following up- and downsides:

  • The model can create as much training data as is needed
  • The model is not bound to real-time
  • The training process can always restart from arbitrary states
  • However, the model still needs real-world data to be trained
  • And an inaccurate model leads to a misbehaving system

Since model-based learning is a mostly internal process, playtime could be extended without distracting the user.

Utilizing these three techniques should enable machine learning for smart home systems. Whether it works in practice, however, will depend on choosing the right parameters. Finding the right parameters usually involves a lot of trail and error. I am eager to find out how much effort it will make to translate my plans into practice.
