Distributing a CNN with Pabiana

Tutorial · Jan 10, 2018

This tutorial describes how to implement a convolutional neural network with Pabiana and TensorFlow. To demonstrate the frameworks hierarchical communication pathways, the network is split into four layers, each of which is run by a different node. Convolutional neural networks are a type of deep neural networks that are inspired by the visual cortex of humans and have been successfully utilized to classify images, sound samples and similar types of data. They make use of the spatial structure of data to reduce complexity and simultaneously gain translation-invariant properties. Convolutional neural networks can be trainined using gradient descent or similar algorithms via backpropagation. Pabiana is a Python framework based on ZeroMQ that allows you to intuitively build and distribute cognitive applications over multiple nodes.

Usually, a trained CNN will be deployed to a single computer, where it can serve as a service to classify incoming data. For this tutorial, however, we will split the network and run each layer in a different process. This example should only act as a demonstration for the variance of different architectures possible with Pabiana. This tutorial will make use of TensorFlow and its Python-3 bindings. Therefore, you should use Pip to install TensorFlow into a virtual environment. Instructions on how to do it can be found on TensorFlow's website. Also, have a look at the other article I wrote on how to install Pabiana. The complete source code of this tutorial can be found in this compressed file. Use the Python interpreter from the virtual environment to run the code.

Deep Network Architecture

The convolutional neural network we will use here is pre-trained to classify handwritten digits from the MNIST dataset. It consists of two convolutional layers, a fully-connected layer and an output layer. The first layer utilizes 32 filters, a RELU activation function and max pooling. The second layer includes 64 filters and the same type of post-processing. The fully-connected layer contains 1024 RELU-activated neurons. The network returns an estimate for each digit, with the largest one to be considered its guess.

area = Area(repo['area-name'], repo['interfaces'])
			config = {

			@area.alteration(source='mnist')
			def mnist_input():
			  if area.context['mnist'][0]['purpose'] == 'prediction':
			    area.publish({
			      'purpose': 'prediction',
			      'data': forward(area.context['mnist'][0]['data']),
			      'label': area.context['mnist'][0]['label']
			    })

The network has been trained as a whole on a single computer with TensorFlow. ADAM optimizer was used as training algorithm. Also, a dropout mechanism was included at the fully-connected layer to reduce overfitting. The weights have been stored as Numpy arrays to a 13 MB file called model.npz.

Pabiana Architecture

The application consists of 4 different areas (mnist, conv, fully, output), each implementing one of the layers. The calculations with TensorFlow for each area have been outsourced to Python modules called predict.

x = tf.placeholder("float")
			weights = tf.Variable(tf.truncated_normal([5, 5, 32, 64], stddev=0.1))
			bias = tf.Variable(tf.constant(0.1, shape=[64]))

			convolution = tf.nn.conv2d(x, weights, strides=[1, 1, 1, 1], padding='SAME')
			relu = tf.nn.relu(convolution + bias)
			graph = tf.nn.max_pool(relu, ksize=[1, 2, 2, 1], strides=[1, 2, 2, 1], padding='SAME')

			model = np.load('./model.npz')

			def forward(data):
			  with tf.Session() as session:
			    session.run(tf.global_variables_initializer())
			    weights.load(model['weights_2'])
			    bias.load(model['bias_2'])
			    data = np.array(data)
			    return graph.eval(feed_dict={x: data}).tolist()

Code extract of convolution/predict.py

Periodically, a picture is randomly chosen from the MNIST testing dataset and fed through the first convolutional layer. The resulting values are then published to the next layer. Eventually, the data has been passed through the whole network. The uppermost output area logs the network's guess to the console and compares it to the real label.

Study the full source code of the tutorial, to figure out how the application works. You can start all the areas with a single command. You also have to run a synchronizing clock togehter with them to make it work. After some time, you will see the data being passed through the hierarchy of areas in the logs. You can compare the network's guesses to the actual labels.

python3 -m pabiana convolution:conv fully:fully mnist:mnist output:output -C

Theoretically, it would be possible to train the network by passing the gradients backwards through the hierarchy of areas. This would, however, extend training time. In practice, Pabiana should be used to connect complex neural networks, not single layers.
