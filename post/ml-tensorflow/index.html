Basic ML with TensorFlow

Tutorial · Jul 22, 2017

TensorFlow is a software library that is mainly used in machine learning. It helps you calculate complex statistical models distributed over multiple CPUs and GPUs. It was created primarily to train and apply deep neural networks, but can also be used in other contexts. Therefore, this article presents three basic concepts of machine learning and explains how to implement them with TensorFlow.

This tutorial will make use of TensorFlow's Python-3 bindings. Therefore, you should use Pip to install TensorFlow into a virtual environment. Instructions on how to do it can be found on TensorFlow's website. The code samples also rely on other libraries like NumPy, Pandas and Matplotlib, so make sure to have them installed. The complete source code of this tutorial, together with the used data, can be found in this compressed file. It contains one script for each of the three examples. Use the Python interpreter from the virtual environment to run the scripts.

Simple Linear Regression

Simple Linear Regression is a model that is based on two-dimensional sample points. These samples consist of an independent (input) variable, the x-coordinate, and a dependent (output) variable, the y-coordinate. The relationship between the coordinates is modeled using a linear predictor function. The function predicts the value of the dependent variable based on the value of the independent variable. There must be some sort of linear relationship between the coordinates for this model to be accurate.

Based on the samples from the training set, the learning algorithm specifies a linear predictor function, the hypothesis hθ. Using this hypothesis, unknown values of y can be estimated for every input value of choice. Since the hypothesis is a univariate linear function, it can be represented by the following formula: hθ: y = θ0 + θ1 · x θ0 and θ1 represent the parameters of the function. The purpose of the learning algorithm is to calculate and optimize these parameters. A cost function J indicates the quality of the currently chosen parameter values. Therefore, the function calculates the average magnitude of error from the distance of each sample point to the hypothesis. J(θ) = 1⁄2m · ∑ ( hθ(x(i)) - y(i) )2 from 1 to m In this equation, m represents the number of samples, while i enumerates each sample. The denominator term 2m is used instead of just m to simplify background calculations. After the differentiable cost function has been defined, the learning algorithm can find parameters that minimize it. Gradient descent, an iterative optimization algorithm, is used to achieve that.

The code for this part can be found in the linear_regression.py file. TensorFlow requires you to declare your formulas in a way it can understand. Therefore, you need to create Variables for the parameters that should be learned and Placeholders for the training values. Besides hypothesis and cost function, the optimization operation must also be declared. Pandas is used to load the training data into a Dataframe. During the learning session, this data is repeatedly fed into the Placeholders. Each time, the parameters get gradually better. At the end, the resulting line is plotted over an image of the data points.

Binary Logistic Regression

In the first example, linear regression predicted real-valued variables along a linear function. In contrast, logistic regression is used for classification. In the binary case, there are two classes, 0 and 1. For each independent variable, logistic regression predicts a y-coordinate between 0 and 1, indicating the class affiliation. There must be a linear separation between the samples of the different classes for this method to work. In this example, the model is based on three-dimensional sample points, one of which specifies the class.

To get values of y between 0 and 1, the hypothesis is a sigmoid function: hθ: y = 1⁄1 + e-t The decision boundary t of this classification method, however, is still a linear function. Since there are two x-values for every sample point, it can be represented by the following formula: t = θ0 + θ1 · x1 + θ2 · x2 After the parameters have been optimized, a value of t(x) ≥ 0, which implicates a value of hθ(x) ≥ 0.5, will be interpreted as affiliation to class 1. Before a learning algorithm can be used to optimize the parameters, again, a cost function is needed. J(θ) = -1⁄m · ∑ [y(i) · log hθ(x(i)) + (1 - y(i)) · log (1 - hθ(x(i)))] from 1 to m Using this convex function, the cost of a correctly classified sample is 0. If the hypothesis, on the other hand, indicates the wrong class, the cost explodes. Again, gradient descent is used to find parameters that minimize the cost function.

The code for this part can be found in the logistic_regression.py file. It works similar to the first example. This time, however, there are two input values, x_1 and x_2. Therefore, we now also have an additional parameter. While we repeated the optimizing step 5000 times in the first example, we now increase that number to 15 000. At the end, the decision boundary is plotted over the image of the data points.

Multilayer Perceptron

In many cases, the classes are not linearly separable. A more complex model is needed for these kinds of classification problems. A multilayer perceptron is a feed-forward neural network model with at least one hidden layer. As in the last example, the samples are points in the two-dimensional space and of two distinct classes. Again, the model predicts a y-coordinate between 0 and 1, indicating the class affiliation.

The multilayer perceptron used here consists of three layers. There are two input units, three hidden units and one output unit. The computation a single hidden unit contributes to the model is similar to the hypothesis of logistic regression. ai = 1⁄1 + e-(θi0 + θi1 · x1 + θi2 · x2) The final hypothesis, computed by the output unit, is also based on a sigmoid function of the combined weighted values of the hidden units. hθ: y = 1⁄1 + e-(θ40 + θ41 · a1 + θ42 · a2 + θ43 · a3) Every directed line in the neural network diagram is linked to one parameter value of θ. Additionally, the θi0 values represent the bias of each unit. The predictions of the hypothesis are interpreted in the same way as with logistic regression. Therefore, the same cost function J can be used together with gradient descent.

The code for this part can be found in the multilayer_perceptron.py file. It works similar to the last example. This time, however, we have a much larger number of parameters — 14. A NumPy array called theta is used to store them all. Instead of 0, they are initialized with a random value. The hypothesis consists of the combined functions of the 4 anterior units. The learning process is repeated 100 000 times, which can take quite a while. At the end, the decision boundary is plotted over the image of the data points as a contour plot. It can be interpreted as the probability of a point placed somewhere on it to belong to one of the classes.
